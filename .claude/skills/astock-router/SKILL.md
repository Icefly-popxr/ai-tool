# A股投研资料路由与分析技能

## 核心功能

智能路由用户输入，自动选择最优分析流程，并按照标准化7列格式生成A股上市公司分析表格，支持知识库增量更新与高亮对比。

## 路由决策树

```
用户输入
    │
    ├─ 包含URL/链接？
    │    │
    │    └─ 是
    │         │
    │         ├─ 调用 agent-reach 提取链接内容
    │         │
    │         ├─ 内容包含A股上市公司？
    │         │    │
    │         │    ├─ 是 → 按7列格式分析输出（等待加入确认）
    │         │    │
    │         │    └─ 否 → 归纳总结，生成.md文档
    │
    ├─ 纯文本/关键词？
    │    │
    │    └─ 是 → 按7列格式分析输出（等待加入确认）
    │
    └─ 用户回复"加入"？
         │
         └─ 是 → 更新知识库 + 生成高亮对比文档
```

## 支持的资料类型

- 网页链接（网页内容抓取）
- 图片（OCR识别）
- PDF文档
- Markdown文档
- 文本内容

## 工作流程

### 场景1：链接输入（网页、抖音、小红书等）

**输入示例：**
```
帮我分析这个视频讲了什么公司：https://v.douyin.com/xxx
```

**处理流程：**
1. **提取内容** - 使用 `agent-reach` skill
   ```bash
   # 抖音视频
   mcporter douyin.parse_douyin_video_info share_link="链接"
   # 通用网页
   curl -s "https://r.jina.ai/URL"
   ```
2. **判断内容类型**
   - 包含A股上市公司 → 继续步骤3
   - 无上市公司 → 跳转场景3
3. **分析内容** - 识别A股上市公司，按7列表格格式输出分析结果，等待用户确认后写入知识库

### 场景2：文本/关键词输入

**输入示例：**
```
帮我分析一下液冷产业链的上市公司
帮我提取这段话里的投资标的
```

**处理流程：**
1. **直接分析** - 识别A股上市公司，按7列表格格式输出分析结果，等待用户确认后写入知识库

### 场景3：链接内容无上市公司

**触发条件：**
- 链接提取的内容中未识别到任何A股上市公司名称
- 内容为：宏观经济分析、行业趋势、新闻事件等非直接投资标的

**处理流程：**
1. **归纳总结**
   - 提取内容核心观点
   - 分析对市场/行业的影响
   - 标注关键数据和结论
2. **生成.md文档**
   - 文件命名：`[YYYYMMDD]_[内容主题].md`
   - 保存路径：`e:\IceFly\TRAE\资料库\`
   - 自动打开文件供用户查看

**文档格式：**
```markdown
# [标题]

## 📅 时间
- 分析时间：[YYYY-MM-DD HH:mm]
- 来源：[平台/作者]

## 📝 核心内容

### 1. 主要观点
[归纳的核心观点]

### 2. 关键数据
[提取的重要数据]

### 3. 影响分析
[对市场/行业的影响]

### 4. 相关公司（间接影响）
[虽然无直接标的，但可能受影响的公司]

## 🔗 原文链接
[原始链接]

## 📊 视频数据（如适用）
- 点赞：xxx
- 评论：xxx
- 收藏：xxx
```

## 输出表格格式（7列）

### 第1列：企业名称
- **必填项**
- A股上市公司名称
- **此列为唯一登记依据** - 只有在此列出现的标的才视为正式登记标的
- 名称需加粗，使用Markdown标记：**公司名**

**示例：**
```markdown
**宁德时代**
```

### 第2列：产业链
使用**申万三级行业标准**进行分类。

**输出要求：**
- 格式：`申万一级-申万二级-申万三级`
- 如：电力设备-电池-电池化学品、电力设备-光伏设备-光伏加工设备

**参考示例：**
```markdown
电力设备-电池-电池化学品
```

**常见产业链参考：**
- 电力设备-电池-电池化学品（璞泰来、亿纬锂能）
- 电力设备-电池-锂电专用设备（赢合科技、先导智能）
- 电力设备-光伏设备-光伏加工设备（晶盛机电、连城数控）
- 电子-半导体-数字芯片设计（寒武纪、景嘉微）
- 汽车-汽车零部件-汽车电子（德赛西威、华阳集团）

### 第3列：细分环节
根据企业的主营业务和产品特点，提炼其细分环节的具体关键词。

**输出要求：**
- 用①-③个具体的技术/产品关键词
- **每个关键词另起一行**
- **不使用完整句子**
- **不使用修饰词**

**示例：**
```markdown
动力电池
储能系统
电池材料
```

### 第4列：战略地位
根据企业在所属产业链中的市场地位、技术水平和行业影响力提炼。

**输出要求：**
- 一句精炼陈述（不超过50字）
- **排名类、地位类、唯一/稀缺类、份额类、突破类关键词需用Markdown加粗**

**加粗关键词类型：**
- 排名类：全球第一、全国领先、龙头等
- 地位类：核心供应商、关键技术等
- 唯一/稀缺类：唯一、独家、首家等
- 份额类：市场份额占比XX%、占比第一等
- 突破类：打破垄断、实现国产替代等

**示例：**
```markdown
全球动力电池装机量**第一**，市场份额超**30%**，**国内唯一**实现CTP技术量产的动力电池企业
```

### 第5列：核心竞争力
**格式要求（精简为1-3句，分号分隔）：**

1. 核心技术/市场要点
2. 国内主要竞争对手（格式：国内主要竞争对手为XX股份、XX科技）
3. 核心客户/合作商（格式：为XX股份等供应XX产品）

**示例：**
```markdown
全球领先的动力电池制造商；国内主要竞争对手为比亚迪、亿纬锂能；为特斯拉、蔚来等供应电池产品
```

### 第6列：新增长级
根据企业的第二增长曲线、新业务布局或跨界拓展方向提炼。

**输出格式：**
1. 先写新增长级定位描述（核心定位词需用Markdown加粗）
2. 再写"关联公司："，列出相关对标的A股公司名称并用Markdown加粗
3. 多个关联公司用顿号（、）分隔
4. 整个列内容用句号分隔不同方向

**示例：**
```markdown
**储能业务**成为第二增长曲线，已实现大型储能项目交付。关联公司：**宁德时代**、**阳光电源**
```

### 第7列：备注（可选）
- 补充说明
- 如资料来源、关键时间节点、风险提示等
- 非必填项

**示例：**
```markdown
2024年储能业务收入同比增长超100%
```

## 内部链接规则（重要）

- 所有提及的公司名称，如果**在第1列"企业名称"中存在**，需加上Markdown锚点链接
- 链接格式：`[**公司名**](#公司名)`
- 如果公司不在知识库中，暂不加链接

**示例：**
如果表格中有"**宁德时代**"和"**比亚迪**"：
```markdown
为**特斯拉**、**蔚来**等供应电池产品
```
（特斯拉不是A股上市公司，不加链接）

```markdown
国内主要竞争对手为**比亚迪**、**亿纬锂能**
```
（比亚迪在第1列，加锚点链接）

**知识库中公司的锚点设置方式：**
在知识库文件中，每个公司信息块前添加对应标题：
```markdown
## 宁德时代 {#宁德时代}

| 企业名称 | 产业链 | ...
```

**注意事项：**
- 锚点名称必须与公司名称一致（不包括特殊字符）
- 锚点链接中不能包含空格，使用`#公司名`格式

## 分析汇总流程

### 1. 接收资料
- 如果是链接，使用WebFetch工具获取内容
- 如果是图片，使用OCR识别（如有工具支持）
- 如果是PDF/Markdown，直接读取内容
- 如果是文本，直接处理

### 2. 分析汇总资料（仅供用户参考，不写入知识库）
对资料进行深度分析，形成汇总报告，包括：
- **资料概要**：资料的来源、类型、主要内容
- **产业链全景**：资料涉及的产业链梳理
- **核心标的**：识别出的主要A股上市公司
- **关键洞察**：产业趋势、竞争格局、技术路线等
- **数据质量评估**：信息的完整性、准确性、可信度

### 3. 识别A股上市公司
- 提取所有提及的A股上市公司名称
- 注意识别股票代码（如000001）和公司全称/简称

### 4. 按列分析
对每个识别的A股上市公司，按7列格式逐一分析：

```
| 企业名称 | 产业链 | 细分环节 | 战略地位 | 核心竞争力 | 新增长极 | 备注 |
|---------|-------|---------|---------|-----------|---------|------|
| **公司名** | 申万一级-申万二级-申万三级 | 关键词1<br>关键词2<br>关键词3 | 精炼陈述（加粗关键词） | 核心要点；竞争对手；核心客户 | 定位描述。关联公司：**公司A**、**公司B** | 可选备注 |
```

### 5. 展示结果
- **先展示分析汇总**（仅供用户参考）
- **再以Markdown表格形式展示分析结果**
- 前置"[待确认]"标记
- 等待用户输入

### 6. 确认并写入知识库
- **用户确认后才写入** - 只有用户明确回复"确认"或"加入"后，才将标的正式写入知识库
- **只写入表格内容** - 分析汇总内容不写入知识库
- 如果用户说"不要"、"删除"等，明确拒绝写入

**用户回复示例：**
- ✅ "加入" - 写入知识库（只写入表格）
- ✅ "确认" - 写入知识库（只写入表格）
- ✅ "好的" - 写入知识库（只写入表格）
- ❌ "不要" - 不写入
- ❌ "算了" - 不写入

## 知识库更新与高亮对比

**触发条件：**
- 用户回复"加入"确认写入知识库
- 本次分析包含**新增上市公司**（原知识库中不存在）

**处理流程：**
1. **对比分析**
   - 读取现有知识库文档
   - 识别本次新增的上市公司
   - 标记更新内容
2. **生成对比文档**
   - 文件命名：`[YYYYMMDD]_知识库更新对比.md`
   - 保存路径：`e:\IceFly\TRAE\知识库对比\`
   - 高亮显示新增/变更内容
3. **更新知识库**
   - 追加新公司到主知识库
   - 使用锚点链接格式

## 错误处理

- **链接提取失败**：提示用户内容无法获取，建议手动复制关键信息
- **无A股公司识别**：自动判断为场景3，生成归纳总结文档
- **部分识别**：只分析识别到的公司，其他跳过
- **知识库冲突**：如公司已存在，提示用户选择覆盖或跳过

## 注意事项

1. **准确性优先** - 只输出有明确资料支撑的信息，不猜测
2. **简洁精炼** - 遵循各列的格式要求，不添加冗余信息
