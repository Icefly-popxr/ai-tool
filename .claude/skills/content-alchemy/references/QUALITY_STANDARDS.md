# 质量标准与常见问题

本文档详细说明口播文案的质量检查标准、常见问题及修复方案。

---

## 质量检查清单

### ✅ **必须有（5项）**

#### 1. 具体的故事或数据
```
❌ 错误："很多人都有拖延症"
✅ 正确："我统计过，73%的拖延是因为'不知道怎么开始'"

❌ 错误："这个方法很有效"
✅ 正确："用这个方法，我的项目完成率从40%提升到了85%"
```

**检查方法**：
- [ ] 有时间/地点/人物的具体信息吗？
- [ ] 有数字/比例/频率吗？
- [ ] 有可感知的细节吗？

#### 2. 核心观点一句话说清
```
❌ 错误："我们要学会时间管理，提高效率，才能更好地完成工作"
✅ 正确："时间管理的本质不是挤时间，是决定不做什么"
```

**检查方法**：
- [ ] 能否在15秒内说清楚？
- [ ] 有认知落差吗？
- [ ] 去词还能表达吗？

#### 3. 可验证的逻辑链
```
❌ 错误："因为你不努力，所以失败"（跳跃因果）
✅ 正确："因为你的目标不清晰（原因）
         → 所以你无法判断优先级（结果）
         → 所以你总是做紧急的事（结果）
         → 所以重要的事永远没时间（最终结果）"
```

**检查方法**：
- [ ] 原因→结果的关系清楚吗？
- [ ] 有中间步骤吗？（不能跳跃）
- [ ] 能找到反例吗？

#### 4. 边界表达
```
❌ 错误："所有人都应该这样做"
✅ 正确："大多数情况下，如果你符合[X条件]，这个方法会有效"

❌ 错误："这个方法永远有效"
✅ 正确："在我测试过的30多个项目中，这个方法在80%的场景下有效"
```

**检查方法**：
- [ ] 用"通常""大多数"代替"永远""所有人"了吗？
- [ ] 说了适用边界吗？
- [ ] 说了可能失效的场景吗？

#### 5. 真实素材
```
❌ 错误：编造案例
❌ 错误：引用来源不明的数据
✅ 正确：使用亲身经历、可查证的公开数据、朋友的真实故事
```

**检查方法**：
- [ ] 案例/数据有来源吗？
- [ ] 是真实发生的吗？
- [ ] 能说出案例的具体细节吗？

---

### ❌ **禁止出现（5项）**

#### 1. 无来源论断
```
❌ "研究表明/专家说/数据显示..."（没有具体来源）
✅ "哈佛商业评论2023年的研究显示..."（有具体来源）
```

#### 2. 鸡汤式空泛语
```
❌ "只要你努力，就一定成功"
❌ "坚持就是胜利"
❌ "相信自己，你可以的"

✅ 替换为有具体内容的表达
```

#### 3. 因果链跳跃
```
❌ "因为你不自律，所以穷"
   缺失中间层：自律→行动→结果→收入

✅ 完整因果链：
"因为你没有清晰的财务目标（原因）
 → 所以你不知道该省哪些钱（结果）
 → 所以你月光（结果）
 → 所以你没有存款（结果）"
```

#### 4. 绝对结论
```
❌ "所有人都应该..."
❌ "这永远是对的"
❌ "只要这样做，就一定能..."

✅ 替换为有边界的表达
```

#### 5. 虚假案例/数据
```
❌ 编造数据："我帮助了10000+人"
❌ 编造经历："有一个学员..."

✅ 只用真实案例和数据
```

## 常见问题与修复方案

### 问题1：开头抓不住人

**症状**：
- 开头太普通，观众划走
- 说了10秒还没进入主题
- 自言自语式的开场

**原因**：
- 上来就讲道理
- 没有制造认知冲突
- 没有用"痛点"开场

**修复方案**：

**方案A：反常识开头**
```
❌ "今天给大家分享一个时间管理的方法"
✅ "你知道吗，你的时间管理方法可能是最大的时间浪费"
```

**方案B：痛点开场**
```
❌ "我想聊聊如何提高效率"
✅ "你是不是也有这种感觉——忙了一天，但什么都没完成？"
```

**方案C：数字冲击**
```
❌ "效率很重要"
✅ "90%的人都在用错误的方式提高效率——包括你"
```

---

### 问题2：PROBLEM段落太长/太空洞

**症状**：
- 讲了一大堆，但观众感受不到"痛"
- 说的都是大家都知道的废话
- 原因分析太浅

**原因**：
- 只描述现象，没有深挖原因
- 没有说"不解决的代价"
- 没有具体细节

**修复方案**：

**三明治法**：
```
第一层（困境）：具体描述现状
第二层（原因）：深刻分析为什么
第三层（代价）：不解决的真实后果
```

**示例**：
```
困境："每次开会，老板问'进度怎么样了'，你就开始紧张"
原因："因为你心里没底——不知道自己做到哪了，还差多少"
代价："结果是：要么你被骂，要么你加班补，要么你撒谎——三个选项都是痛苦"
```

---

### 问题3：故事讲得干巴巴

**症状**：
- 故事没有画面感
- 听起来像流水账
- 没有情感共鸣

**原因**：
- 缺少细节
- 没有冲突
- 没有转折

**修复方案**：

**故事填充法**：

| 要素 | 问题 | 填充 |
|------|------|------|
| 时间 | 什么时候？ | "那天是凌晨3点" |
| 地点 | 在哪？ | "我坐在办公室" |
| 人物 | 有谁？ | "只有我和电脑" |
| 动作 | 做了什么？ | "我盯着屏幕发呆" |
| 冲突 | 卡在哪了？ | "但就是不想动" |
| 转折 | 怎么突破的？ | "直到我做了X" |
| 结果 | 后来呢？ | "从那以后..." |

---

### 问题4：观点太空泛

**症状**：
- 观点听起来有道理，但不知道怎么做
- 道理太正确，没有洞见
- 一句话说不清

**原因**：
- 没有提炼
- 没有找到核心
- 用词太保守

**修复方案**：

**观点提炼三问**：
1. "去掉哪个字，这句话就不完整了？"
2. "有认知落差吗？听众听到会不会意外？"
3. "能直接指导行动吗？"

**示例**：
```
原观点："我们要学会合理安排时间"
提炼后："不是时间不够，是你把优先级排错了"

原观点："要保持积极的心态"
提炼后："积极不是假装开心，是找到值得开心的理由"

原观点："要多读书学习"
提炼后："不是读得不够，是读完没用起来"
```

---

### 问题5：行动建议太大

**症状**：
- 观众听完"嗯，有道理"，但不会行动
- 建议太复杂，需要很多步骤
- 没有降低行动门槛

**原因**：
- 没有把行动拆到最小
- 没有给"成功标志"
- 没有移除借口

**修复方案**：

**最小行动法**：
```
问："观众现在能做的最小动作是什么？"
再问："还能更小吗？"
再问："这个动作5分钟内能开始吗？"
```

**示例**：
```
❌ "从今天开始养成每天学习的习惯"
✅ "今天睡前花3分钟，在手机备忘录里写下明天最重要的1件事"

❌ "要学会管理自己的情绪"
✅ "下次想发火的时候，先深呼吸3次——就这3次，其他的之后再说"
```

---

### 问题6：结尾太弱

**症状**：
- 结尾突然，没有力量
- 说"谢谢大家"很突兀
- 没有回应开头

**原因**：
- 把结尾当成了结束
- 没有形成闭环
- 没有情感调动

**修复方案**：

**闭环收尾法**：
```
开头问的问题 → 结尾给出答案
开头的悬念 → 结尾揭晓
开头的主张 → 结尾重复+强化
```

**示例**：
```
开头："为什么你总是计划完不成？"
结尾："所以，计划完不成不是因为你懒，
       而是你的计划从一开始就注定失败。
       下次做计划，记住——少即是多。"
```

## 时间版本调整指南

### 3分钟版本

**核心原则**：只讲一件事，讲透

**结构建议**：
- 开场：15秒，反常识/痛点
- PROBLEM：30秒，一个核心困境
- 故事：45秒，一个完整故事
- 观点：15秒，一句话说清
- 行动：10秒，一个最小行动
- 结尾：5秒，闭环即可

**砍掉**：对标迁移（太长）

### 5分钟版本

**核心原则**：有深度，有广度

**结构建议**：
- 开场：20秒
- PROBLEM：45秒，可用铺垫递进（两个问题）
- 故事：90秒，核心故事+简短案例
- 观点：20秒
- 对标：30秒，1-2个类比
- 行动：15秒
- 结尾：10秒

### 7分钟+版本

**核心原则**：建立权威感

**结构建议**：
- 开场：25秒
- PROBLEM：60秒，深挖问题根源
- 故事：120秒，2个故事或有深度的单一故事
- 观点：30秒
- 对标：60秒，2-3个类比
- 行动：20秒
- 结尾：15秒

## 质量自检流程

### 输出前检查

```
1. 开场有没有认知冲突？
   → 没有：重写开场

2. PROBLEM有没有说"代价"？
   → 没有：加入不解决的真实后果

3. 故事有没有具体细节？
   → 没有：填充时间/地点/数字/冲突

4. 观点能不能一句话说清？
   → 不能：提炼核心，删除废话

5. 行动够不够小？
   → 不够：拆分到"5分钟内能开始"

6. 有没有边界表达？
   → 没有：加入"通常""大多数""可能"

7. 结尾有没有闭环？
   → 没有：回应开场
```

### 输出后检查

```
1. 读出声来通顺吗？
   → 不通顺：调整语序

2. 语速适合吗？（3分钟约450字）
   → 太快/太慢：调整长度

3. 有没有"呃""那个""然后"的口语病？
   → 有：删除或替换

4. 观众能跟上吗？
   → 不能：简化逻辑，增加停顿
```
