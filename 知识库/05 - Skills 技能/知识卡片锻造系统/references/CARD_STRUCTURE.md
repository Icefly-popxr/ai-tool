# 卡片结构详解

本文档详细说明知识卡片的每个字段的含义、填写要求和示例。

## 完整卡片模板

```markdown
🏷️ 卡片名称：[简短命名]
📅 创建/更新：[日期]
🔖 卡片ID：KC-[领域缩写]-[序号]
📊 等级：L[1-3]
💡 核心洞察（一句话）
   [用最精炼的语言表达这张卡的知识要点]
✍️ 行动指南
   [具体怎么用、步骤或决策规则]
🟢 绿灯（适用场景）
   · [场景1]
   · [场景2]
🔴 红灯（不适用场景）
   · [场景1]
   · [场景2]
🔗 关联卡片
   - [关系] KC-XXX: [说明]
🌐 关联链接
   - [官方网页] [链接标题](URL)
   - [内部笔记] [[笔记名称]]
   - [书籍] 《书名》 → 第X章
```

## 字段详解

### 🏷️ 卡片名称

**要求**：简短、精准、便于记忆

**命名规则**：
- 使用2-6个汉字
- 优先使用动词开头（如"先写成果再写计划"）
- 或使用主语+动作结构（如"领导反馈好的周报结构"）

**良好示例**：
- ✅ "周报结构：成果优先"
- ✅ "先倾听再表达"
- ✅ "用具体数字代替模糊表述"

**不良示例**：
- ❌ "关于周报写作的一些建议"（太长）
- ❌ "沟通技巧"（太模糊）
- ❌ "ABC法则"（除非用户已熟悉该缩写）

### 🔖 卡片ID

**格式**：`KC-[领域缩写]-[序号]`

**领域缩写建议**：
- `WORK` - 职场工作
- `COMM` - 沟通协作
- `LEARN` - 学习方法
- `PROD` - 生产效率
- `MGMT` - 管理决策
- `CREA` - 创造力
- `LIFE` - 生活智慧

**序号规则**：
- 同领域内按时间顺序编号
- 从01开始，不足两位补零

**示例**：
- `KC-WORK-01`
- `KC-COMM-03`
- `KC-LEARN-12`

### 📊 等级

**含义**：卡片的成熟度和验证程度

- **L1（初铸）**：通过四铁律检验的初始版本
- **L2（试炼）**：已在2个真实场景中验证有效
- **L3（淬火）**：跨越2个以上维度差异的深度验证

详见 [UPGRADE_CRITERIA.md](UPGRADE_CRITERIA.md)

### 📜 type

**定义**：知识的类型分类

**可选类型**：
- **人物**：关于特定人物的认知和判断方法
- **工具**：如何使用某个工具或软件
- **概念**：抽象概念的理解和应用
- **比较**：两个或多个选项的对比分析
- **模型**：可复用的思维框架或流程
- **技术**：具体的技术实现方法
- **模式**：反复出现的问题解决方案
- **方法**：做事的方式和步骤

**选择建议**：
- 如果侧重"判断人"→ 人物
- 如果侧重"用什么做"→ 工具
- 如果侧重"是什么/为什么"→ 概念
- 如果侧重"选哪个好"→ 比较
- 如果侧重"怎么想/怎么推演"→ 模型
- 如果侧重"怎么实现"→ 技术
- 如果侧重"这类问题都这么办"→ 模式
- 如果侧重"怎么做"→ 方法

### 💡 核心洞察

**要求**：一句话说清知识要点

**判断标准**：
1. 读完这句话，能指导一个具体行动
2. 不需要额外解释就知道怎么用
3. 去掉任何一个字都会损失含义

**撰写技巧**：
- 用"要...而不是..."结构（如：要先说结论，而不是绕弯子）
- 用"当...时，应该..."结构（如：当对方情绪激动时，应该先共情再给方案）
- 用动词开头（如：先...再...；用...代替...）

**示例对比**：
```
❌ "沟通很重要，需要技巧"
✅ "当需要说服他人时，先说利益点而不是讲道理"

❌ "要善于倾听"
✅ "当对方表达观点时，先复述确认再发表意见"
```

### ✍️ 行动指南

**要求**：具体到可以直接执行

**判断标准**：
- 用户看完可以直接操作，不需要再思考"怎么做"
- 包含明确的步骤或决策规则
- 避免模糊词汇（"适度"、"适量"、"适当"）

**撰写结构**：
1. **步骤型**：第一步... → 第二步... → 第三步...
2. **条件型**：当[条件]时，执行[操作]
3. **决策型**：如果[情况]，选择[方案A]；如果[情况]，选择[方案B]

**良好示例**：
```
✅ "汇报工作时，按以下顺序：成果 → 困难 → 计划"
✅ "收到邮件后，24小时内必须回复；如果无法解决，说明预计解决时间"
✅ "会议开始前，先问'今天的会议想达成什么结果？'"
```

**不良示例**：
```
❌ "要灵活运用沟通技巧"（太模糊）
❌ "适度表达情绪"（什么叫适度？）
❌ "做好时间管理"（太笼统）
```

### 🟢 绿灯 & 🔴 红灯

**绿灯（适用场景）**：这张卡有效的情形

**红灯（不适用场景）**：这张卡失效或慎用的情形

**填写要求**：
1. 必须来自真实经历，不接受推测
2. 每张卡至少1个绿灯 + 1个红灯
3. 绿灯和红灯要成对出现（什么情况下有效，什么情况下无效）

**常见配对模式**：

| 绿灯场景 | 红灯场景 |
|---------|---------|
| 对方有决策权 | 对方只是传话筒 |
| 一次性任务 | 需要持续协作 |
| 时间充裕 | 时间紧迫 |
| 正式场合 | 非正式场合 |
| 老员工 | 新人 |
| 熟人 | 陌生人 |
| 问题清晰 | 问题模糊 |
| 执行层面 | 战略层面 |

**判断原则**：
- 红灯不能为空，也不能写"所有情况都适用"
- 红灯要具体，不是"特殊情况"这种废话
- 绿灯范围要准确，不能过度夸大

**示例**：
```
🟢 绿灯：
· 向上级汇报工作进展时
· 向客户介绍项目成果时
· 写述职报告时

🔴 红灯：
· 写日记或私人笔记时（不需要结构化）
· 问题还在摸索阶段时（还没结论）
· 需要保密的工作内容（不能写具体成果）
```

### 🔗 关联卡片

**格式**：`[关系类型] KC-ID: [说明]`

**关系类型**：
- **补充**- 补充这张卡的内容
- **对比**- 与这张卡形成对比
- **依赖**- 使用这张卡的前置条件
- **延伸**- 这张卡的进阶应用
- **替代**- 可替代这张卡的方案

**示例**：
```
🔗 关联卡片
- 补充 KC-WORK-03: "KC-WORK-03提供了更详细的汇报框架"
- 对比 KC-COMM-07: "KC-COMM-07强调情感，此卡强调逻辑"
- 依赖 KC-LEARN-01: "要先掌握KC-LEARN-01的学习方法"
```

### 🌐 关联链接

**用途**：记录卡片的来源或延伸阅读，方便回溯验证

**来源类型**：
- **官方网页** - 可直接访问的网页链接
- **内部笔记** - 知识库内的其他笔记（Obsidian `[[]]` 格式）
- **书籍** - 纸质书或电子书，标注出处章节
- **论文/文章** - 学术论文或长文
- **视频** - 视频资源链接

## 快速检查清单

生成卡片前，逐项检查：

- [ ] 卡片名称是否简短精准？
- [ ] ID格式是否正确？
- [ ] 核心洞察能否一句话说清？
- [ ] 行动指南能否直接执行？
- [ ] 绿灯是否来自真实经历？
- [ ] 红灯是否具体可操作？
- [ ] 关联链接是否已填写来源？
- [ ] 是否通过四铁律检验？
