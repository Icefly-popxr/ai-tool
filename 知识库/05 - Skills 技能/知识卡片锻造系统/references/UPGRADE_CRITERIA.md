# 卡片升级标准

本文档详细说明知识卡片的等级定义和升级流程。

## 等级定义

### L1 - 初铸

**含义**：通过四铁律检验的初始版本

**特征**：
- ✅ 通过原子性检验
- ✅ 通过可行动检验
- ✅ 通过可证伪检验（红灯不为空）
- ✅ 通过诚实性检验（绿灯来自真实经历）
- ✅ 至少1个绿灯场景
- ✅ 至少1个红灯场景

**局限性**：
- 仅来自单一场景或经历的提炼
- 适用范围可能未经充分验证
- 行动指南可能需要进一步打磨

**默认态度**：
- 维持L1是默认态度
- 用户需主动提供证据才能升级

---

### L2 - 试炼

**含义**：已在2个真实场景中验证有效

**升级条件**：
- ✅ 已经是L1
- ✅ 用户提供2个以上的真实使用经历
- ✅ 这2个场景有明显差异

**场景差异维度**：

| 维度 | 示例 |
|------|------|
| **对象差异** | 面对不同的人（上级vs平级vs下属） |
| **环境差异** | 场景不同（线上vs线下） |
| **利害差异** | 重要性不同（日常汇报vs晋升答辩） |
| **频率差异** | 频率不同（首次vs多次） |

**示例**：

✅ **有效升级**：
```
场景1：向CEO汇报季度成果（对象差异）
场景2：向团队成员布置周任务（对象差异）
→ L1升级为L2
```

❌ **无效升级**：
```
场景1：向A领导汇报
场景2：向B领导汇报（同一对象类型，不算差异）
→ 不能升级
```

**升级验证要求**：
- 用户必须提供具体经历
- 不接受"可能能用"、"应该可以"
- 必须能说出效果如何

**升级话术**：
```
"想升级到L2，需要你提供至少2个真实使用经历。
比如：你在什么场景下用过这个方法？
效果怎么样？"
```

---

### L3 - 淬火

**含义**：跨越2个以上维度差异的深度验证

**升级条件**：
- ✅ 已经是L2
- ✅ 用户提供跨越2个以上维度的使用经历
- ✅ 跨维度验证通过

**维度跨越示例**：

✅ **有效升级**：
```
L2场景：
- 向CEO汇报（对象：上级）
- 线上向团队布置任务（环境：线上）

L3新场景：
- 向董事会汇报（对象：高层+战略层面）→ 跨越对象维度
- 线下跨部门协调会议（环境：线下+多方） → 跨越环境维度
→ L2升级为L3
```

**跨维度要求**：
- 至少2个维度有明显差异
- 每个维度内的场景要有明显区分
- 需要提供每个维度的具体使用案例

**升级验证要求**：
- 必须有跨越维度的证据
- 不接受同一维度内的重复验证
- 需要说明不同维度下的效果差异

**升级话术**：
```
"想升级到L3，需要跨越2个以上维度的验证。
你已经验证了对象差异（不同层级的汇报）和环境差异（线上和线下）。
还有其他维度的使用经历吗？
比如：在利害关系更重要的场景下效果如何？"
```

## 升级流程

### 触发条件

1. **用户主动申请**
   - 用户输入：`/升级 [卡片ID]`
   - AI不能主动建议升级

2. **用户提供证据**
   - 用户描述新的使用经历
   - AI评估是否符合升级条件

3. **AI验证**
   - 检查场景差异是否足够
   - 检查证据是否具体
   - 检查是否满足维度要求

### 验证检查清单

**L1 → L2 验证**：
- [ ] 是否已有L1等级？
- [ ] 是否提供至少2个真实场景？
- [ ] 场景之间是否有明显差异？
- [ ] 效果如何？（不接受"应该有用"）
- [ ] 是否跨越至少1个维度？

**L2 → L3 验证**：
- [ ] 是否已有L2等级？
- [ ] 是否提供跨越2个以上维度的证据？
- [ ] 每个维度的场景是否足够区分？
- [ ] 跨维度的效果是否有差异？
- [ ] 是否能说明维度变化带来的挑战？

### 升级失败处理

**不能升级的情况**：
- 场景数量不够
- 场景之间没有差异
- 证据不具体（"应该有用"不算）
- 还在同一维度内

**处理方式**：
```
我理解你对这个方法有信心。
但根据L2的升级标准，需要至少2个有明显差异的真实场景。
你现在提供的场景：
- 场景1：...
- 场景2：...

这两个场景的差异还不够明显。
能否再提供一个不同维度（比如对象、环境、利害等）的使用经历？
```

### 降级处理

**触发条件**：
- 用户提供的新证据表明原有红灯场景不完整
- 用户发现原有绿灯场景不准确

**处理方式**：
- 修正红灯/绿灯描述
- 维持当前等级，但标注需要更新
- 不能降级（等级只升不降）

## 升级决策树

```
用户申请 /升级 [卡片ID]
    ↓
检查当前等级
    ↓
┌─────────────────┐
│ 等级 = L1       │
│ 验证L2条件      │
│ 场景≥2？        │
│ 差异明显？      │
└─────────────────┘
    ↓是
┌─────────────────┐
│ 等级 = L2       │
│ 验证L3条件      │
│ 维度≥2？        │
│ 跨维度证据？    │
└─────────────────┘
    ↓是
┌─────────────────┐
│ 等级 = L3       │
│ 已达最高        │
│ 不能继续升级    │
└─────────────────┘

场景<2 → 维持L1，提供升级建议
差异不明显 → 维持L1，说明原因
维度<2 → 维持L2，说明原因
```

## 特殊升级情况

### 卡片合并

当用户想把两张L2卡片合并时：
- 合并后的卡片继承最高的等级
- 如果等级相同，合并为L2
- 需要重新验证四铁律

### 卡片拆分

当一张卡片包含多个知识点时：
- 拆分成多张L1卡片
- 每张独立验证升级
- 不能继承原卡片的等级

## 升级记录

每次升级需要记录：
- 升级时间
- 原等级 → 新等级
- 触发升级的场景/证据
- 升级验证人（用户确认）
