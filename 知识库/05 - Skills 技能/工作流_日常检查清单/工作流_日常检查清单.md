# 日常检查清单工作流

> 定期检查知识库健康状况，确保持续迭代

---

## 📋 检查节奏

| 频率 | 检查内容 | 耗时 |
|------|----------|------|
| **每日** | 收件箱清零 + 日记回顾 | 5分钟 |
| **每周** | 全面巡检 + 归档清理 | 15分钟 |
| **每月** | 知识库健康报告 + 概念库增长 | 30分钟 |

---

## 一、每日检查（下班前5分钟）

### 1.1 收件箱清零
检查 `00 - Inbox 灵感库/` 中的待处理内容：
- 有明确方向 → 写成知识卡片
- 是项目相关 → 移到 `01 - Projects 项目/`
- 是长期话题 → 移到 `02 - Areas 长期关注/`
- 暂时不重要 → 移到 `04 - Archive 归档/`
- 纯粹垃圾 → 删除

### 1.2 日记回顾
对AI说：

```
回顾今天的日记，提取1个值得写成知识卡片的内容
```

---

## 二、每周检查（周末15分钟）

### 2.1 全库巡检

对AI说：

```
帮我做本周知识库巡检：
1. 统计各个目录的文件数量变化
2. 检查 Inbox 是否有积压
3. 本周新增了多少知识卡片
4. 本周新增了多少概念到 Wikis
5. 有没有超过30天未处理的归档内容
```

### 2.2 归档清理
检查 `04 - Archive 归档/`：
- 超过30天的内容 → 确认是否真的不需要了
- 仍需要的 → 移回对应目录继续完善
- 彻底无用的 → 删除

### 2.3 RSS内容处理
检查 `03 - Resources 参考资料/articles/` 中的新文章：

对AI说：

```
扫描 Resources 中本周新增的文章，识别其中的关键概念，提取到 Wikis 概念库
```

---

## 三、每月检查（30分钟深度复盘）

### 3.1 知识库健康报告

对AI说：

```
生成知识库月度健康报告：
1. 各目录文件数量和变化趋势
2. 本月新增概念数和总概念数
3. 哪个Skill使用频率最高
4. 知识周转率（入库 vs 归档 vs 删除）
5. 发现的问题和改进建议
```

### 3.2 概念库增长
检查 `06 - Wikis 概念库/concepts/` 的增长：
- 本月新增了多少概念
- 哪些概念有连接关系
- 是否需要做概念碰撞

### 3.3 项目进度
检查 `01 - Projects 项目/`：
- 各项目的完成度
- 有没有长期停滞的项目
- 是否要关停或归档

### 3.4 知识库Lint深度检查（追加）

> 让AI做一次跨页面质量检测，发现隐藏问题

对AI说：

```
做知识库Lint深度检查：
1. 跨页面矛盾检测 — 有没有两张卡片说相反的话
2. 孤儿页检测 — 哪些页面没有入链引用
3. 过期内容检测 — 有没有被新内容覆盖的老观点
4. 缺失引用 — 重要概念提了但没有独立Wiki页面
5. 缺口发现 — 哪些地方可以web搜索补充资料
6. 更新 index.md 和 log.md — 同步最新状态
```

> lint 结果应同步追加到 `log.md` 中
