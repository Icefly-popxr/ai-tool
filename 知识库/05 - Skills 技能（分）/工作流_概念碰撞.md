# 概念碰撞工作流

> 把两个看似无关的概念放在一起，产生新的认知洞见
> 参考：胡子哥「认知匕首 3.0」方法论

---

## 📋 完整流程

```
① 从Wikis概念库中取出一个关键概念
      （如：控制环悖论、同群效应、反脆弱...）
       ↓
② 从Inbox/文章/卡片中取当前素材
      （如：身份维护、流量规模...）
       ↓
③ 计算概念关联度（阈值：0.6-0.75）
      ↓
      不是随机，是有一定关联但不完全相同
       ↓
④ 概念碰撞 → 产生新认知
      ↓
⑤ 评估产出：
      ├─ 可注卡 → 写成知识卡片
      ├─ 可升华 → 扩展到 Areas
      └─ 可反馈 → 回到灵感库继续迭代
```

---

## 🚀 触发方式

### 方式一：指定两个概念
对AI说：

```
概念碰撞：[概念A] × [概念B]
```

**示例**：
```
概念碰撞：身份维护 × 控制环悖论
```

AI会返回碰撞分析，格式如下：

```
━━━ 概念碰撞报告 ━━━

输入概念：
  · 身份维护（来自：流量规模）
  · 控制环悖论（来自：控制论）

关联度：0.68（有效碰撞）

碰撞产出：
[新认知洞见]

推导链：
[逻辑链条]

建议操作：
□ 注卡 → 写入 Areas
□ 扩展 → 进一步研究
□ 存档 → 留待后续
```

### 方式二：随机碰撞
对AI说：

```
随机概念碰撞：从Wikis中随机取2个概念做碰撞
```

---

## 🎯 碰撞参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 相似度阈值 | 概念间的关联度下限 | 0.6 |
| 相似度上限 | 太相似则无新意 | 0.75 |
| 产出评估 | 碰撞后需评估是否可用 | 必选 |

> 阈值太低 → 产生垃圾连接  
> 阈值太高 → 找不到碰撞对象  
> 0.6-0.75 是"有一定关联但不完全相同"的最佳区间

---

## 📝 碰撞示例

**输入组合**：身份维护 × 控制环悖论

**碰撞产出**：
> 如果执行层面过度占用了控制层的带宽，控制就会丧失动机，导致系统失去和现实的价值连接。

**推导链**：
```
身份维护不是"带宽的占用"
→ 它是一种反馈环的寄生
→ 流量越多，越享受流量带来的好处
→ 被裹挟，创造更多流量
→ 陷入流量的陷阱
→ 系统为了追求反馈稳定性而自我收缩
```
