# 追问技巧指南

本文档提供锻造师在追问过程中的技巧和话术模板。

## 追问原则

### 核心原则
1. **不替用户思考** - 用户说到的才算，没说的不脑补
2. **每次问一个** - 一次问太多会混乱
3. **等用户回答** - 不连续追问，给用户反应时间
4. **用用户语言** - 复述用户的表述，不擅自改写

### 什么时候追问

```
用户说 → 需要追问 → 追问内容
──────────────────────────────────
"这个很有用" → ❌ → 具体怎么用？
"效果很好" → ❌ → 和之前比有什么不同？
"我学会了" → ❌ → 用在哪里了？
"领导反馈好" → ✅ → 是哪个领导？什么场合？
```

## 锻造流程追问

### 第一阶段：提炼核心洞察

**目标**：将用户的经验压缩成一句话

**追问话术**：

```
1. "能用一句话说清这个经验的核心吗？"
2. "如果只记住一点，你会记住什么？"
3. "这句话去掉哪些词还能表达同样的意思？"
4. "如果要给一个没听过的人解释，你会怎么说？"
```

**检查清单**：
- [ ] 核心洞察是否在15字以内？
- [ ] 是否包含具体的行动？
- [ ] 是否去掉了"有时候"、"可能"等模糊词？
- [ ] 是否可以直接指导行为？

### 第二阶段：构建行动指南

**目标**：将洞察转化为可执行的步骤

**追问话术**：

```
1. "具体是怎么做的？第一步做什么？"
2. "有标准化的步骤吗？"
3. "做的时候有什么注意事项？"
4. "如果要让别人也能用，你会告诉他哪几点？"
```

**检查清单**：
- [ ] 是否有明确的步骤？
- [ ] 能否直接照着执行？
- [ ] 是否有判断标准？
- [ ] 是否避免了"适度"、"适量"等词？

### 第三阶段：标定绿灯场景

**目标**：明确卡片适用的真实场景

**追问话术**：

```
1. "你在什么情况下用过这个方法？"
2. "最近一次是什么时候用的？"
3. "还有哪些情况也能用？"
4. "有什么共同特点吗？"
```

**检查清单**：
- [ ] 场景是否来自真实经历？
- [ ] 是否有具体的时间、地点、对象？
- [ ] 是否描述了实际效果？
- [ ] 场景之间是否有差异？

### 第四阶段：标定红灯场景

**目标**：明确卡片失效的场景

**追问话术**：

```
1. "什么情况下这个方法不管用？"
2. "有没有失败过的情况？"
3. "有没有遇到问题的时候？"
4. "谁用了可能会出问题？"
```

**检查清单**：
- [ ] 红灯是否具体可操作？
- [ ] 是否是真实经历，不是推测？
- [ ] 是否与绿灯形成对比？
- [ ] 是否能帮助用户判断是否适用？

## 四铁律检验追问

### 铁律一：原子性

**问题**：
- "这张卡能否拆成两张？"
- "是否包含多个独立知识点？"
- "是否需要前置知识才能理解？"

**通过标准**：
- 一卡一识
- 独立完整
- 无需前置

### 铁律二：可行动

**问题**：
- "读完这张卡，能直接做什么？"
- "行动指南是否明确？"
- "有什么判断标准吗？"

**通过标准**：
- 有明确的行动
- 有具体的步骤
- 有判断依据

### 铁律三：可证伪

**问题**：
- "什么情况下这张卡会失效？"
- "红灯场景能说清吗？"
- "有人用了会出问题吗？"

**通过标准**：
- 红灯不为空
- 红灯具体明确
- 能帮助判断边界

### 铁律四：诚实性

**问题**：
- "绿灯场景是真实用过的吗？"
- "还是推测出来的？"
- "有没有扩大适用范围？"

**通过标准**：
- 绿灯来自真实经历
- 不夸大适用范围
- 红灯来自真实教训

## 迭代锻造追问

### 场景更新

**追问话术**：
```
1. "新经验是什么？"
2. "和之前有什么不同？"
3. "效果变了吗？"
4. "需要调整行动指南吗？"
```

### 边界探索

**追问话术**：
```
1. "这次用和上次有什么不同？"
2. "同样的方法用在不同对象上，效果一样吗？"
3. "环境变化会影响效果吗？"
```

### 升级评估

**追问话术**：
```
1. "能举一个使用这个方法的例子吗？"
2. "效果如何？"
3. "和之前的方法相比，有什么不同？"
4. "还有其他使用场景吗？"
5. "这些场景之间有什么差异？"
```

## 常见错误避免

### ❌ 错误：连续追问
```
用户：我用这个方法效果很好。

AI：什么效果？用在谁身上？什么场景？什么时候开始的？

问题：一次问太多，用户不知道怎么回答
✅ 正确：一次问一个，等用户回答后再问下一个
```

### ❌ 错误：脑补补充
```
用户：这个方法很有用。

AI：我理解是在工作中用，能帮助提高效率，对吗？

问题：没等用户确认就擅自补充
✅ 正确：追问"能举个例子吗？"而不是直接脑补
```

### ❌ 错误：模糊追问
```
用户：有时候要注意。

AI：要注意什么？

问题："什么"太模糊，用户不知道怎么回答
✅ 正确："在什么情况下要注意？" - 问具体场景
```

### ❌ 错误：引导性追问
```
用户：我觉得...

AI：是不是像你说的那样...

问题：把用户没说的意思强加给用户
✅ 正确：让用户自己说，不做假设
```

## 追问话术模板库

### 开场引导
- "能举个例子吗？"
- "具体是怎么做的？"
- "什么时候开始的？"
- "对谁用的？"

### 深入细节
- "能说得更具体些吗？"
- "当时是什么情况？"
- "后来怎么样了？"
- "和之前相比有什么不同？"

### 确认理解
- "我理解你的意思是...，对吗？"
- "是这样吗？"
- "还有其他情况吗？"
- "我复述一下你看对不对..."

### 探索边界
- "什么情况下不管用？"
- "有没有遇到问题？"
- "有没有失败过？"
- "谁用了可能会出问题？"

### 提炼总结
- "一句话总结，你会怎么说？"
- "如果只记住一点，是什么？"
- "最重要的经验是什么？"

## 追问检查清单

每次追问前，检查：

- [ ] 这个问题是否只有一个答案？
- [ ] 用户能直接回答吗？
- [ ] 需要用户思考还是回忆？
- [ ] 是否避免了引导性提问？
- [ ] 是否有足够的上下文？

## 用户确认的重要性

### 必须确认的场景
1. 核心洞察是否准确
2. 行动指南是否完整
3. 绿灯场景是否真实
4. 红灯场景是否准确
5. 卡片是否需要调整

### 确认话术
```
"我理解你意思是..."
"是这样吗？"
"你说的...是指..."
"我复述一遍，你看对不对..."
```

### 不确认的后果
- 用户可能默认同意，但实际不是那个意思
- 卡片可能偏离用户真实意图
- 用户无法真正使用这张卡
- 失去卡片的可信度
