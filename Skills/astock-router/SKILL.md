# A股投研资料路由技能

## 核心功能

智能路由用户输入，自动选择最优分析流程，并支持知识库增量更新与高亮对比。

## 路由决策树

```
用户输入
    │
    ├─ 包含URL/链接？
    │    │
    │    └─ 是
    │         │
    │         ├─ 调用 agent-reach 提取链接内容
    │         │
    │         ├─ 内容包含A股上市公司？
    │         │    │
    │         │    ├─ 是 → 调用 astock-analysis 分析
    │         │    │
    │         │    └─ 否 → 归纳总结，生成.md文档
    │
    ├─ 纯文本/关键词？
    │    │
    │    └─ 是 → 调用 astock-analysis 分析
    │
    └─ 用户回复"加入"？
         │
         └─ 是 → 更新知识库 + 生成高亮对比文档
```

## 工作流程

### 场景1：链接输入（网页、抖音、小红书等）

**输入示例：**
```
帮我分析这个视频讲了什么公司：https://v.douyin.com/xxx
```

**处理流程：**
1. **提取内容** - 使用 `agent-reach` skill
   ```bash
   # 抖音视频
   mcporter douyin.parse_douyin_video_info share_link="链接"

   # 通用网页
   curl -s "https://r.jina.ai/URL"
   ```

2. **判断内容类型**
   - **包含A股上市公司** → 继续步骤3
   - **无上市公司** → 跳转场景3

3. **分析内容** - 使用 `astock-analysis` skill
   - 识别A股上市公司
   - 按7列表格格式输出分析结果
   - 等待用户确认"加入"后写入知识库

### 场景2：文本/关键词输入

**输入示例：**
```
帮我分析一下液冷产业链的上市公司
帮我提取这段话里的投资标的
```

**处理流程：**
1. **直接分析** - 使用 `astock-analysis` skill
   - 识别A股上市公司
   - 按7列表格格式输出分析结果
   - 等待用户确认"加入"后写入知识库

### 场景3：链接内容无上市公司

**触发条件：**
- 链接提取的内容中未识别到任何A股上市公司名称
- 内容为：宏观经济分析、行业趋势、新闻事件等非直接投资标的

**处理流程：**
1. **归纳总结**
   - 提取内容核心观点
   - 分析对市场/行业的影响
   - 标注关键数据和结论

2. **生成.md文档**
   - 文件命名：`[YYYYMMDD]_[内容主题].md`
   - 保存路径：`e:\IceFly\TRAE\资料库\`
   - 自动打开文件供用户查看

**文档格式：**
```markdown
# [标题]

## 📅 时间
- 分析时间：[YYYY-MM-DD HH:mm]
- 来源：[平台/作者]

## 📝 核心内容

### 1. 主要观点
[归纳的核心观点]

### 2. 关键数据
[提取的重要数据]

### 3. 影响分析
[对市场/行业的影响]

### 4. 相关公司（间接影响）
[虽然无直接标的，但可能受影响的公司]

## 🔗 原文链接
[原始链接]

## 📊 视频数据（如适用）
- 点赞：xxx
- 评论：xxx
- 收藏：xxx
```

### 场景4：知识库更新与高亮对比（新增）

**触发条件：**
- 用户回复"加入"确认写入知识库
- 本次分析包含**新增上市公司**（原知识库中不存在）

**处理流程：**
1. **对比分析**
   - 读取现有知识库文档
   - 识别本次新增的上市公司
   - 标记更新内容

2. **生成对比文档**
   - 文件命名：`[YYYYMMDD]_知识库更新对比.md`
   - 保存路径：`e:\IceFly\TRAE\知识库对比\`
   - 高亮显示新增/变更内容

3. **更新知识库**
   - 追加新公司到主知识库
   - 使用锚点链接格式

## 高亮对比功能详解

### 高亮标记规则

| 标记类型 | 语法 | 用途 |
|---------|------|------|
| **新增公司** | `==**公司名**==` | 首次加入知识库的公司 |
| **更新内容** | `==更新内容==` | 某列内容有变化的部分 |
| **新增内容** | `++新增内容++` | 某列新增的描述 |

**注意：**
- 新增公司需要高亮区分
- 更新内容也需要高亮标记
- 同一公司可能有多列更新，都需要标记

## 使用示例

### 示例1：抖音链接（有上市公司）
**用户输入：**
```
分析这个视频的投资标的：https://v.douyin.com/KrE2qmKb1ZQ/
```

**系统处理：**
1. 调用 agent-reach 提取视频内容
2. 识别出薄膜铌酸锂产业链上市公司
3. 调用 astock-analysis 生成7列分析表格
4. 等待用户确认

**用户确认：**
```
加入
```

**系统处理：**
1. 检查知识库中是否已有这些公司
2. 识别新增公司（知识库中不存在）
3. 生成高亮对比文档
4. 更新主知识库（新增公司用`==**公司名**==`标记）

### 示例2：抖音链接（无上市公司）
**用户输入：**
```
分析这个视频：https://v.douyin.com/xxx
```

**系统处理：**
1. 调用 agent-reach 提取视频内容
2. 未识别到A股上市公司
3. 归纳总结核心内容
4. 生成.md文档：`[日期]_视频标题.md`
5. 提示用户文档已生成

### 示例3：网页链接
**用户输入：**
```
帮我分析这篇文章：https://example.com/article
```

**系统处理：**
1. 调用 agent-reach 获取网页内容
2. 判断是否包含A股公司
3. 有公司 → 调用 astock-analysis
4. 无公司 → 生成.md文档

**用户确认：**
```
加入
```

**系统处理：**
1. 生成高亮对比文档
2. 更新知识库

### 示例4：纯文本
**用户输入：**
```
分析一下光模块产业链的核心标的
```

**系统处理：**
1. 调用 astock-analysis 直接分析
2. 根据知识库识别相关上市公司
3. 生成分析表格

## 错误处理

- **链接提取失败**：提示用户内容无法获取，建议手动复制关键信息
- **无A股公司识别**：
  - 自动判断为场景3
  - 生成归纳总结文档
  - 不进入astock-analysis流程
- **部分识别**：只分析识别到的公司，其他跳过
- **知识库冲突**：如公司已存在，提示用户选择覆盖或跳过

---

## 注意事项

1. **抖音视频限制**：语音转文字API有文件大小限制（<25MB），大视频可能失败
2. **登录内容**：小红书等平台需要登录的内容无法抓取
3. **实时性**：市场热点变化快，建议注明分析时间
4. **知识库确认**：分析结果必须用户明确回复"加入"后才写入
5. **无上市公司处理**：优先生成归纳文档，而非直接丢弃内容
6. **高亮对比**：只高亮标记更新内容，新增公司无需特殊标记
